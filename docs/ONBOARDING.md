# Customer Onboarding

## Step 1 — Choose an artifact

The customer chooses one artifact:

- AI-agent output trace;
- software release;
- benchmark result;
- math/proof support file;
- compliance workflow;
- dataset transformation record.

## Step 2 — Define the claim

Use one narrow sentence:

```text
This receipt verifies that [artifact] passed [declared checks] under [policy/version].
```

## Step 3 — Add evidence

Attach:

- SHA-256 hashes;
- command outputs;
- test logs;
- allowed file paths;
- timestamps;
- version tags;
- links to public repository files when available.

## Step 4 — Run verifier

Use browser, CLI, or API.

## Step 5 — Publish certificate

Publish only the verified claim, never a global overclaim.
